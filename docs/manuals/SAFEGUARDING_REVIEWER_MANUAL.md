# Safeguarding reviewer manual

Safeguarding review is a separate audited evidence projection. It never enrolls the reviewer in a conversation and must not change participant access, receipts, unread counts, or notifications.

1. Search using the minimum safe metadata needed.
2. Choose the correct reason category and give a case-specific justification.
3. Acknowledge protected access and start the bounded review session.
4. Review participants, timeline, media, receipts, flags, restrictions, moderation history, and existing notes.
5. Add a flag or append-only note only when it advances the authorised review.
6. Use restriction/closure/tombstone controls only when authorised by school procedure.
7. Request an export only for an approved purpose; verify its ready/verified state before download.
8. End the review promptly.

Do not copy protected evidence into ordinary messaging. Do not disclose confidential reasons to participants. If access appears broader than expected, stop, record the correlation/reference, and use the incident runbook.
