# Safeguarding administrator guide

Safeguarding gives specifically authorized school staff a protected way to find and
review school conversations. Access is time-limited, reason-gated and audited. Having
a teacher or school-administrator role does not grant safeguarding access by itself.

## Find and review a conversation

1. Open **School → Safeguarding → Message reviews**.
2. Start with **Student** or **Participant**. Add the date range and conversation status
   when needed. **More filters** contains participant type, class/grade, branch,
   message type, direction and flagged/restricted/closed states.
3. Select **Search**. Results show conversation metadata, never message previews.
4. Choose **Review** on the intended student or staff conversation. Use **Details** only
   if you need the internal conversation ID.
5. Choose the real reason category, write a specific justification of at least 15
   meaningful characters, acknowledge audited access and choose the review duration.
6. Select **Start review**. The dedicated review page shows the expiry time and has no
   message composer.

Use protected photo and voice controls only inside the active review. Moderation,
internal notes, flags, exports and audit history are grouped separately from the
message timeline. Every dangerous change asks for confirmation and a confidential
reason. End the review when the work is complete.

## Manage safeguarding access

Only an authorized access manager can open **Safeguarding → Permission management**.

1. Select the staff member.
2. Review the single checklist of current safeguarding access.
3. Check or clear only the access required for that person's duties.
4. Enter the reason for the access change.
5. Select **Save access**.

**Manage safeguarding access** lets a person grant or remove access for others, so
grant it only when that responsibility is intended. All access is scoped to the
selected school; never use one school's access decision as authority for another.

## Mobile and privacy notes

On a phone, filters and actions stack within the screen, and **More filters** keeps the
common search short. Android Back closes the review sheet, protected photo or
confirmation before leaving the page. Arabic uses the same workflow in RTL.

Do not copy internal notes into ordinary messages, share an active review URL, or
upload an evidence package to a public service. If access, receipts, unread counts or
notifications change merely because a review was opened, stop and report it as an
incident using the safeguarding operations runbook.
